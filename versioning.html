Le Versionning

Une fois qu'une première version du projet à été réaliser, il est désormait temps de me faire évoluer.
Pour ce faire on vas réaliser plusieurs version d l'application ou du site. Ces version peuvent majeur ou mineur.
Elle peuvent être due à l'ajout d'une nouvelle fonction ou à une faille de sécurité, issue ausi bien d'un montée de version ou d'une vulnérabilité.
Dans tout les cas il faut maintir sont code a jour. user style sheets.

On met ainsi en place un verionning.
On utilisera le plus souvent git pour enregistrer différente version du code.
Voici la bonne pratique pour enregistrer une nouvelle version. 

        
          # Initialisation d'un repo git
          git init
          
          # Ajouter les modifications
          git add .
          
          # Enregistrer les modifications
          git commit -m"Un message indiquant les modifications apportées à chaque fichier"

Attention: ces modificaion serront uniquement enregistrer localement sur la branch git active (voir doc)
Dans le cadre d'un application ou d'un site, on poussera ces modification vers un dépot en ligne.
De cette manière on peut non seulement mettre en commun le travail, mais aussi publier les différentes version de manière succives tout en concervant les anciennes.

Cependant: on ne pousse pas se modification n'importe commun au risque d'effacer le travail commun.
Après avoir commit ses modificaiton en locale ont :

  • Se met à jour avec le dépot à distance
  • Résouds les conflits existants
  • Alors seuelement on pousse ses modifications
          
            # Se mettre à jour avec le repo en ligne
            git pull origin BranchDeTravail

            # On vérifie que l'on est à jour et on résouds les conlfit.
            git push origin BranchDeTravail
