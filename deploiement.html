Introduction au Déploiement

Le déploiement est le processus permettant de rendre une application web accessible aux utilisateurs après son développement.
Il assure l’accessibilité, la stabilité et la sécurité du projet pour les utilisateurs finaux.

Les Étapes du Déploiement

1. Pré-déploiement

Avant de mettre en ligne, le code est mis à jour et validé via un commit Git. Cette étape assure que le projet est prêt pour le déploiement.

Outils : GitHub, GitLab.

2. Étape de Build

Installation des dépendances et compilation du code, assurant qu'il fonctionne avec les bibliothèques et frameworks utilisés.

Outils : Docker, Webpack, Babel.

3. Tests

Les tests unitaires, d'intégration et de performance sont exécutés pour détecter d'éventuels bugs avant la mise en production.

Outils : PHPUnit, Jest, Cypress, Selenium.

4. Préproduction

Le code est déployé sur un environnement de test permettant de valider le bon fonctionnement en conditions réelles.

Outils : Docker, Kubernetes, Netlify.

5. Déploiement en Production

Si toutes les étapes précédentes sont validées, le code est déployé en production. Le déploiement peut être progressif pour éviter les risques.

Stratégies de Déploiement

1 - Déploiement progressif : Mise en ligne par palier (10% → 50% → 100%).
2 - Blue-Green Deployment : Deux versions tournent en parallèle pour éviter les interruptions.
3 - Rolling Update : Mise à jour progressive sans downtime.

Outils : CI/CD : GitHub Actions, GitLab CI/CD, Jenkins. Hébergement : AWS, Google Cloud, OVH, Netlify.

6. Surveillance et Maintenance

Une fois l’application en ligne, il est crucial de surveiller ses performances.
Des outils comme New Relic et Datadog permettent d'identifier les erreurs et d’optimiser le site.

Vérifier la stabilité et la rapidité du site.
Surveiller les logs pour détecter les anomalies.
Effectuer des tests de charge (ex: JMeter).

7. Rollback : Gestion des Incidents

En cas de problème après le déploiement, un rollback permet de restaurer la version précédente.
Cela garantit la stabilité du site.

Annulation de la mise à jourAnnulation de la mise à jour si un problème critique est détecté.
Restauration des fichiers et bases de données sauvegardées.
Analyse des erreurs et correction avant un nouveau déploiement.

Conclusion

Ce qu’il faut retenir :
1- Un déploiement bien organisé garantit une expérience utilisateur fluide.
2- Les tests et la préproduction sont des étapes incontournables.
3- Automatiser avec CI/CD permet d'éviter les erreurs humaines.
4- Surveiller en continu pour détecter les incidents à temps.
5- Avoir toujours un plan de rollback en cas de problème critique.

Clé du succès : Tester, automatiser, surveiller, corriger !
